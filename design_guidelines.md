# design_guidelines.md

## 设计规范文档

本文档是《面向小学生信息科技课程的模拟在线购物网站》项目的统一设计规范。所有参与本项目开发的 AI 代理、设计实现者和项目维护者必须严格遵循本文档的设计要求。

***

## 1. 设计目标

### 1.1 视觉与交互设计目标

| 目标 | 说明 |
|-----|------|
| 清晰易懂 | 页面结构简单，信息层级分明，小学生一眼能看懂 |
| 友好温暖 | 色彩明亮柔和，不刺眼，给人安全、舒适的感觉 |
| 操作简单 | 交互路径短，点击次数少，每一步都有明确反馈 |
| 教学适用 | 适合课堂投屏演示，重点信息突出，便于教师讲解 |
| 数据可观察 | 状态变化明显，学生能看到数据如何变化 |

### 1.2 设计定位

```
本项目是「教学型模拟购物网站」，不是「商业电商网站」

设计定位 = 教育产品 + 儿童友好 + 简化购物流程演示

核心差异：
- 商业电商：追求转化率、促销感、信息密度高
- 本项目：追求可理解、可演示、流程清晰

设计风格：
- 不是淘宝/京东风格
- 不是低幼卡通风格
- 是「简洁明亮的教育产品风格」
```

### 1.3 设计如何服务教学场景

| 教学场景 | 设计要求 |
|---------|---------|
| 教师课堂演示 | 页面结构清晰，重点信息突出，投屏效果好 |
| 学生独立操作 | 按钮大、文字清晰、操作简单、反馈明显 |
| 理解购物流程 | 流程步骤清晰，状态变化可见，数据变化可观察 |
| 阅读代码学习 | 界面不过度复杂，代码与界面对应关系清晰 |

## 2. 设计原则

### 2.1 简洁清晰

| 原则含义 | 具体要求 |
|---------|---------|
| 信息精简 | 每个页面只展示必要信息，不堆砌内容 |
| 结构简单 | 页面结构扁平，层级不超过 3 层 |
| 视觉干净 | 留白充足，不使用过多装饰元素 |
| 文字简短 | 按钮文字不超过 4 个字，提示文字不超过 20 个字 |

### 2.2 小学生友好

| 原则含义 | 具体要求 |
|---------|---------|
| 按钮够大 | 最小点击区域 44px × 44px |
| 文字够大 | 正文不小于 16px，标题不小于 20px |
| 颜色柔和 | 不使用高饱和度刺眼颜色 |
| 图标易懂 | 使用常见、易懂的图标，不使用抽象符号 |
| 文字简单 | 使用小学生能理解的词汇，避免专业术语 |

### 2.3 教学优先

| 原则含义 | 具体要求 |
|---------|---------|
| 重点突出 | 关键操作和状态使用主色或强调色 |
| 流程可见 | 购物流程的每个步骤都有明确标识 |
| 状态清晰 | 订单状态、支付状态用颜色和文字双重表达 |
| 数据可见 | 购物车数量、订单金额等数据变化明显 |

### 2.4 操作反馈明显

| 原则含义 | 具体要求 |
|---------|---------|
| 即时反馈 | 点击按钮后立即有视觉变化 |
| 成功提示 | 操作成功后显示绿色成功提示 |
| 错误提示 | 操作失败后显示红色错误提示和原因 |
| 加载提示 | 数据加载时显示加载动画 |
| 确认操作 | 重要操作（删除、提交）需二次确认 |

### 2.5 信息层级明确

| 层级 | 用途 | 视觉表现 |
|-----|------|---------|
| 一级 | 页面标题、主要操作 | 最大字号、主色按钮 |
| 二级 | 内容标题、次要操作 | 中等字号、次色按钮 |
| 三级 | 正文内容、辅助信息 | 标准字号、灰色文字 |

### 2.6 可扩展但不过度设计

| 原则含义 | 具体要求 |
|---------|---------|
| 结构清晰 | 模块划分清晰，便于后续添加功能 |
| 样式统一 | 使用 CSS 变量，便于整体调整 |
| 不过度 | 不预留未确定功能的 UI 位置 |
| 不复杂 | 不使用复杂的 UI 模式和交互方式 |

## 3. 整体视觉风格

### 3.1 风格关键词

```
简洁、明亮、友好、清晰、温暖、教育感
```

### 3.2 页面整体气质

| 气质特征 | 说明 |
|---------|------|
| 明亮温暖 | 使用明亮的背景色和柔和的主色 |
| 干净整洁 | 留白充足，不拥挤 |
| 友好亲切 | 圆角设计，柔和的阴影 |
| 专业可信 | 不是低幼风格，而是简洁的教育产品风格 |

### 3.3 风格定位说明

```
✅ 推荐风格：
- 类似教育类产品的简洁风格
- 类似儿童友好网站的明亮风格
- 类似简化版购物网站的功能风格

❌ 避免风格：
- 淘宝/京东的商业促销风格
- 低幼卡通产品的幼稚风格
- 极简冷淡的成人风格
- 花哨炫酷的游戏风格
```

### 3.4 应避免的风格

| 避免风格 | 具体表现 | 为什么避免 |
|---------|---------|-----------|
| 商业促销风 | 大量 Banner、促销标签、倒计时 | 不符合教学定位，干扰学习 |
| 低幼卡通风 | 过于可爱的卡通形象、大量装饰 | 不适合 8-10 岁学生 |
| 极简冷淡风 | 黑白灰配色、极细线条 | 不够友好温暖 |
| 花哨炫酷风 | 复杂动画、渐变、特效 | 分散注意力，影响教学 |

## 4. 色彩规范

### 4.1 主色

| 颜色名称 | 色值 | 用途 | 说明 |
|---------|------|------|------|
| 主色 | `#4A90D9` | 主按钮、链接、选中状态 | 友好的蓝色，不刺眼，有信任感 |
| 主色浅 | `#6BA8E8` | Hover 状态 | 主色变亮 10% |
| 主色深 | `#3A7BC8` | Active 状态 | 主色变深 10% |

**主色选择理由**：
- 蓝色是教育类产品常用色，给人信任、稳定的感觉
- 这个蓝色饱和度适中，不刺眼，适合长时间观看
- 适合课堂投屏，投影效果清晰

### 4.2 辅助色

| 颜色名称 | 色值 | 用途 | 说明 |
|---------|------|------|------|
| 辅助色 | `#67C23A` | 成功状态、确认按钮 | 清新的绿色，表示成功 |
| 辅助色浅 | `#85D462` | Hover 状态 | - |
| 辅助色深 | `#5AAF30` | Active 状态 | - |

### 4.3 功能色

## 5. 字体与排版规范

### 5.1 字体选择

| 类型 | 字体 | 说明 |
|-----|------|------|
| 主字体 | `-apple-system, BlinkMacSystemFont, "Segoe UI", Roboto, "Helvetica Neue", Arial, sans-serif` | 系统默认字体，保证可读性 |
| 备用字体 | `"PingFang SC", "Microsoft YaHei", "Hiragino Sans GB", sans-serif` | 中文系统字体 |

**不使用特殊字体**：
- 不引入外部字体文件（加载慢）
- 不使用艺术字体（可读性差）
- 使用系统默认字体保证兼容性和加载速度

### 5.2 字号层级

| 层级 | 字号 | 用途 | 行高 |
|-----|------|------|------|
| H1 页面标题 | 28px | 页面主标题 | 1.3 |
| H2 区块标题 | 24px | 内容区块标题 | 1.4 |
| H3 卡片标题 | 20px | 卡片、模块标题 | 1.4 |
| H4 小标题 | 18px | 列表项标题 | 1.5 |
| 正文 | 16px | 正文内容 | 1.6 |
| 辅助文字 | 14px | 描述、说明、标签 | 1.5 |
| 小字 | 12px | 时间、备注 | 1.4 |

**小学生可读性要求**：
- 正文不小于 16px（比常规网站大）
- 重要信息不小于 18px
- 按钮文字不小于 16px

### 5.3 字重规范

| 字重 | 数值 | 用途 |
|-----|------|------|
| 常规 | 400 | 正文内容 |
| 中等 | 500 | 小标题、强调 |
| 粗体 | 600 | 页面标题、按钮 |

**不使用过粗字重**：
- 不使用 700 以上字重（太粗不好看）
- 不使用 300 以下字重（太细不清晰）

### 5.4 行高规范

| 内容类型 | 行高 | 说明 |
|---------|------|------|
| 标题 | 1.3 - 1.4 | 标题行高较小，更紧凑 |
| 正文 | 1.6 | 正文行高较大，便于阅读 |
| 按钮文字 | 1.2 | 按钮文字紧凑 |
| 输入框文字 | 1.5 | 输入框文字适中 |

### 5.5 文字密度要求

| 要求 | 说明 |
|-----|------|
| 段落间距 | 至少 16px |
| 行间距 | 使用 1.6 行高 |
| 字间距 | 默认，不调整 |
| 段落长度 | 每行不超过 50 个汉字 |

## 6. 布局规范

### 6.1 页面宽度

| 设置项 | 数值 | 说明 |
|-------|------|------|
| 最大宽度 | 1200px | 内容区域最大宽度 |
| 最小宽度 | 320px | 支持的最小宽度 |
| 内容居中 | 居中显示 | 内容区域居中 |
| 左右边距 | 16px（移动端）/ 24px（桌面端） | 内容与屏幕边缘的距离 |

### 6.2 栅格与卡片布局

```
推荐布局方式：

1. 使用 Flexbox 布局（简单易懂）
2. 不使用复杂的 CSS Grid
3. 商品列表使用卡片网格布局
4. 表单使用垂直堆叠布局

卡片网格：
- 首页推荐商品：4 列（桌面）/ 2 列（平板）
- 商品列表页：4 列（桌面）/ 3 列（平板）/ 2 列（移动）
- 卡片间距：16px - 24px
```

### 6.3 模块留白规则

| 区域 | 留白 | 说明 |
|-----|------|------|
| 页面顶部 | 0 | Header 固定在顶部 |
| 页面底部 | 32px | 底部留白 |
| 区块之间 | 24px - 32px | 区块间距 |
| 卡片内部 | 16px | 卡片内边距 |
| 表单项之间 | 16px | 表单项间距 |
| 按钮之间 | 8px - 16px | 按钮间距 |

### 6.4 页面组织方式

```
页面结构：

┌─────────────────────────────────────┐
│           顶部导航栏 (Header)          │
│  Logo | 搜索框 | 购物车 | 用户信息      │
├─────────────────────────────────────┤
│                                     │
│                                     │
│            主体内容区域               │
│         (最大宽度 1200px)            │
│                                     │
│                                     │
└─────────────────────────────────────┘

顶部导航栏：
- 高度：60px
- 背景：白色
- 底部边框：1px 浅灰色
- 固定在顶部（可选）

主体内容区域：
- 背景：浅灰色（#F5F7FA）
- 内边距：24px
```

## 9. 交互反馈规范

### 9.1 点击反馈

| 元素 | 反馈方式 |
|-----|---------|
| 按钮 | 背景色变化 + 轻微缩放 |
| 卡片 | 轻微上浮 + 阴影加深 |
| 链接 | 文字颜色变化 |
| 标签 | 背景色变化 |

```
实现方式：
- Hover 状态：背景色变亮或变深
- Active 状态：轻微缩放（scale: 0.98）
- 过渡时间：0.2s
```

### 9.2 Hover / Active 状态

| 状态 | 视觉表现 |
|-----|---------|
| Hover | 背景色变化、边框颜色变化、阴影变化 |
| Active | 轻微缩放、背景色加深 |
| Focus | 边框变为主色、显示聚焦环（可选） |
| Disabled | 灰色背景、灰色文字、鼠标禁用样式 |

### 9.3 表单校验提示

| 场景 | 提示方式 |
|-----|---------|
| 输入为空 | 输入框红色边框 + 下方红色提示文字 |
| 格式错误 | 输入框红色边框 + 下方红色提示文字 |
| 密码不一致 | 输入框红色边框 + 下方红色提示文字 |
| 用户名已存在 | 输入框红色边框 + 下方红色提示文字 |

```
提示文字示例：
- "请输入用户名"
- "密码长度至少 6 位"
- "两次密码输入不一致"
- "用户名已存在"
```

### 9.4 操作成功提示

| 操作 | 提示内容 | 提示类型 |
|-----|---------|---------|
| 注册成功 | "注册成功，请登录" | 成功 |
| 登录成功 | "登录成功" | 成功 |
| 加入购物车 | "已加入购物车" | 成功 |
| 提交订单 | "订单提交成功" | 成功 |
| 支付成功 | "支付成功" | 成功 |
| 确认收货 | "确认收货成功" | 成功 |
| 提交评价 | "评价提交成功" | 成功 |

### 9.5 操作失败提示

| 操作 | 提示内容 | 提示类型 |
|-----|---------|---------|
| 登录失败 | "用户名或密码错误" | 错误 |
| 注册失败 | "注册失败，请重试" | 错误 |
| 提交失败 | "提交失败，请重试" | 错误 |

### 9.6 "点了就有反应"原则

```
所有可点击元素必须有反馈：

1. 按钮
   - Hover：背景色变化
   - Active：轻微缩放
   - 点击后：执行操作 + 显示结果提示

2. 链接
   - Hover：颜色变化
   - 点击后：跳转或执行操作

3. 卡片
   - Hover：上浮效果
   - 点击后：跳转详情

4. 表单提交
   - 点击后：显示加载状态
   - 完成后：显示成功/失败提示
```

## 14. 动效与过渡规范

### 14.1 是否需要动画

```
第一版动画策略：

✅ 可以有：
- 按钮点击反馈（背景色变化）
- 页面切换过渡（淡入淡出）
- Toast 提示动画（滑入滑出）
- Hover 状态过渡

❌ 不需要：
- 复杂的入场动画
- 元素滚动动画
- 加载骨架屏动画
- 复杂的交互动画
```

### 14.2 可以添加动画的地方

| 位置 | 动画类型 | 时长 |
|-----|---------|------|
| 按钮 Hover | 背景色过渡 | 0.2s |
| 页面切换 | 淡入淡出 | 0.3s |
| Toast 提示 | 滑入滑出 | 0.3s |
| 弹窗显示 | 淡入 | 0.2s |
| 数量变化 | 数字变化 | 无 |

### 14.3 不要添加动画的地方

| 位置 | 原因 |
|-----|------|
| 商品列表加载 | 增加复杂度，影响性能 |
| 价格变化 | 干扰阅读 |
| 状态变化 | 干扰理解 |
| 表单验证 | 分散注意力 |

### 14.4 动画原则

```
动画设计原则：

1. 简单
   - 使用简单的过渡效果
   - 不使用复杂的 keyframe 动画

2. 快速
   - 动画时长不超过 0.3s
   - 不影响操作响应

3. 自然
   - 使用 ease 或 ease-out 缓动
   - 不使用弹跳效果

4. 可预测
   - 动画效果符合预期
   - 不使用意外的动画方向
```

## 17. 执行摘要

### 设计定位

本项目是**教学型模拟购物网站**，设计风格为**简洁明亮的教育产品风格**，不是商业电商风格。

### 核心设计原则

1. **简洁清晰**：信息精简，结构简单
2. **小学生友好**：按钮大、文字大、颜色柔和
3. **教学优先**：重点突出，流程可见
4. **操作反馈明显**：每个操作都有明确反馈

### 色彩方案

- 主色：#4A90D9（友好蓝色）
- 成功色：#67C23A（绿色）
- 警告色：#E6A23C（橙色）
- 错误色：#F56C6C（红色）
- 背景：#F5F7FA（浅灰色）

### 字号规范

- 页面标题：28px
- 区块标题：24px
- 卡片标题：20px
- 正文：16px（最小）
- 辅助文字：14px

### 组件规范

- 按钮最小高度：44px
- 输入框高度：44px
- 卡片圆角：8px
- 使用 CSS 变量统一管理

### 禁止事项

- 不使用商业电商风格
- 不添加促销营销元素
- 不使用深色主题
- 不添加复杂动画
- 不超出第一版功能范围

### 验收标准

- 符合色彩、字号、间距规范
- 适合小学生使用（按钮大、文字大）
- 适合课堂演示（投屏清晰、流程明确）
- 不过度设计（功能、动画、样式简洁）
