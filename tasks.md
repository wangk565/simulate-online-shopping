# tasks.md

## AI 协同开发任务清单

本文档是《面向小学生信息科技课程的模拟在线购物网站》项目的任务清单。所有参与本项目开发的 AI 代理应按照本文档的任务列表逐项推进开发。

***

## 1. 任务说明

### 1.1 文档用途

本文档用于：

- 将第一版开发内容拆分成**具体、可执行、可验收**的任务
- 为 AI 开发代理提供明确的任务指引
- 确保开发过程**小步迭代、逐项完成、逐项验证**
- 防止范围蔓延和过度设计

### 1.2 如何使用任务清单

| 使用者 | 使用方式 |
|-------|---------|
| AI 开发代理 | 按任务编号顺序执行，每完成一个任务输出报告 |
| 项目维护者 | 对照验收标准检查任务完成情况 |
| 代码审查者 | 根据任务范围审查代码改动 |

### 1.3 AI 任务推进粒度

```
推荐粒度：

✅ 每轮完成 1 个任务
✅ 或完成 1 组强相关的小任务（不超过 3 个）
✅ 每轮改动不超过 5 个文件
✅ 单文件新增代码不超过 200 行

❌ 不要一次完成多个不相关任务
❌ 不要一次改动超过 5 个文件
❌ 不要在未完成当前任务时开始下一个任务
```

***

## 2. 任务拆解原则

### 2.1 核心原则

| 原则 | 说明 |
|-----|------|
| 任务要小 | 单个任务可在 1-2 小时内完成 |
| 任务要明确 | 任务目标清晰，不模糊 |
| 任务要可完成 | 任务有明确的产出 |
| 任务要可验证 | 任务有明确的验收标准 |
| 任务要独立 | 任务之间依赖关系明确 |

### 2.2 任务拆分规则

```
正确拆分：

✅ T01: 初始化 Vue 3 项目
✅ T02: 配置 Vue Router
✅ T03: 创建商品静态数据

错误拆分：

❌ T01: 完成项目初始化和所有配置（太大）
❌ T02: 实现用户模块（太大，应拆分为多个任务）
❌ T03: 优化代码（太模糊，没有明确目标）
```

### 2.3 任务执行顺序原则

```
必须遵循的顺序：

1. 先基础、后业务
   - 先完成项目骨架、路由、存储等基础设施
   - 再开发用户、商品等业务模块

2. 先核心、后边缘
   - 先完成主流程必需的功能
   - 再完善交互体验

3. 先数据、后页面
   - 先准备数据源和存储
   - 再开发页面展示

4. 先功能、后优化
   - 先实现功能逻辑
   - 再优化用户体验
```

## 8. 执行摘要

### 任务清单用途

本文档将第一版开发拆分为 **50 个具体任务**，AI 开发代理应按任务编号顺序逐项执行。

### 任务执行原则

1. **小步迭代**：每轮完成 1 个任务或 1 组强相关小任务
2. **逐项验证**：每个任务完成后必须验证验收标准
3. **输出报告**：每个任务完成后必须输出完成报告
4. **不超范围**：严格按照任务内容执行，不添加额外功能

### 任务分组

- G1-G2: 基础设施（P0）
- G3-G4: 核心业务（P1）
- G5-G6: 流程功能（P2）
- G7-G8: 完善功能（P3-P4）
- G9: 优化体验（P5）

### 关键约束

- 不引入真实后端、真实支付、真实物流
- 不添加 PRD 范围外的功能
- 不引入技术栈外的依赖
- 页面能看不等于任务完成，必须可交互、可验证

### 验收标准

所有任务完成后，主购物流程应可完整走通：注册/登录 → 浏览商品 → 加入购物车 → 提交订单 → 模拟支付 → 查看订单 → 确认收货 → 评价商品
